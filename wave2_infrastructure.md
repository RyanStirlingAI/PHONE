# Wave 2: Political Infrastructure Systems

## The operational backbone of the How Many NZ campaign

This document specifies the five infrastructure systems that turn a solo operator with Claude Code into a functioning political intelligence operation. Each system is designed to run on zero budget, to compound over time, and to feed content into the pipeline defined in wave1_content.md.

---

## 1. THE OIA PIPELINE

The Official Information Act is the campaign's single most valuable tool. It provides exclusive content, forces government transparency, creates accountability stories even when agencies refuse to comply, and gives journalists material they can publish without needing to verify independently. A well-run OIA operation is the difference between being a social media commenter and being a political intelligence source.

### 1.1 Legal Framework

Every OIA request must comply with the Official Information Act 1982. The key provisions:

- Section 12: Any person (citizen, resident, or otherwise) can request official information from any government agency or Minister of the Crown.
- Section 15: The agency must respond within 20 working days. They can extend this under section 15A if the request is large or requires consultation, but must notify you of the extension within the original 20 days.
- Section 6: Grounds for refusal that are conclusive (national security, legal privilege, etc.). Rarely applicable to immigration policy requests.
- Section 9: Grounds for refusal that require a balancing test against the public interest. This is where most fights happen. The agency must demonstrate that the reason for withholding outweighs the public interest in release.
- Section 18: Administrative refusals (request too broad, information doesn't exist, etc.).

### 1.2 Request Template

Every request follows a standard structure. Claude Code generates the specific content; the legal framing stays consistent.

```
[Your name or organisation name]
[Email address]
[Date]

To: [Agency name]
By email: [OIA email address]

Dear [Agency],

I am writing to request, under the Official Information Act 1982, the following information:

[Specific request -- see section 1.3 for the 30 requests]

To assist with your processing of this request, I am seeking:
- Any briefings to Ministers (BIMs) on this topic
- Any aide-memoires or internal analysis documents
- Any Cabinet papers or committee papers related to this topic
- Any relevant data sets or statistical tables

If any information is withheld, I request that you provide the specific section of the OIA under which it is withheld, and your reasoning for why the public interest in withholding outweighs the public interest in release.

If this request cannot be fulfilled as stated, I would appreciate the opportunity to refine it before a formal decision is made, as provided for under section 18B of the Act.

I would prefer to receive the information electronically, in the format in which it is held.

Thank you for your assistance.

[Your name]
```

### 1.4 The Follow-Up Process

The follow-up system runs on rigid timelines. Claude Code tracks every request and flags required actions automatically.

**Day 1:** Request filed. Logged in the tracking system with: agency, date sent, subject, 20-working-day deadline, email confirmation reference.

**Day 10 (half-time check):** If no acknowledgement has been received, send a brief follow-up email: "I am writing to confirm receipt of my OIA request dated [date] regarding [subject]. I would be grateful for confirmation that the request is being processed."

**Day 20 (deadline):** If no response or extension notification has been received, the request is overdue. Send a formal follow-up: "My OIA request dated [date] regarding [subject] is now overdue. Under the OIA, agencies are required to respond within 20 working days. I request an immediate update on the status of this request."

**Day 25 (five days overdue):** If still no response, send a notice of complaint: "I note that my OIA request dated [date] remains unanswered, now five working days past the statutory deadline. If I do not receive a response within five further working days, I will be lodging a complaint with the Office of the Ombudsman regarding the delay."

**Day 30 (ten days overdue):** File a complaint with the Ombudsman. The complaint itself becomes content: "I asked [agency] for [information] 30 working days ago. They've blown through the legal deadline. I've now asked the Ombudsman to investigate. What are they hiding?"

### 1.5 Handling Refusals and Partial Releases

Every type of response generates a different content opportunity.

**Full release:** The best outcome. Extract key findings, create OIA Reveal content (see wave1_content.md, Pillar 3). Send the full documents to relevant journalists with a one-paragraph summary.

**Partial release with redactions:** Highlight what was released. Then highlight what was withheld and the stated reason. Content angle: "They told us X. They won't tell us Y. Here's why that matters." If the withholding reason is section 9 (the balancing test), challenge it: the public interest in knowing the government's own analysis of immigration impacts is high, particularly in an election year.

**Full refusal under section 9:** Appeal to the Ombudsman. Content angle: "The government has this analysis but won't share it. If the news was good, they'd release it." The refusal itself is often more newsworthy than the information would have been.

**Refusal as "too broad" (section 18):** Resubmit with narrower scope. This is not a defeat. It's the agency telling you what they hold. Narrow the request based on their guidance and resubmit immediately. Content angle: "I refined my request. Let's see if they cooperate this time."

**"Information doesn't exist" response:** This is often the most powerful finding of all. "I asked the Ministry of Education whether they track how many school places are needed because of migration. They told me they don't. The government adds 130,000 people a year and nobody counts the classrooms." The absence of information is a planning failure, which is the campaign's core argument.

### 1.6 The Ombudsman Appeal Process

The Ombudsman is the enforcement mechanism for the OIA. Appeals are free and can be submitted by email.

**When to appeal:**
- Agency fails to respond within 20 working days (plus any notified extension)
- Information is withheld and you believe the public interest in release outweighs the reason for withholding
- The agency has extended the deadline unreasonably
- The agency has refused your request as "too broad" without offering to help you refine it (section 18B requires them to assist)

**How to appeal:** Email info@ombudsman.parliament.nz with: the original request, the agency's response (or lack thereof), the specific grounds for your complaint, and your argument for why the information should be released.

**What happens:** The Ombudsman investigates. This can take weeks to months. The investigation itself creates pressure on the agency, because the Ombudsman can formally recommend release and the agency must comply or seek a Cabinet veto (which is extremely rare and politically damaging). The Ombudsman publishes case notes which can be cited.

**Strategic use:** File Ombudsman complaints proactively. Even if you don't expect to win every complaint, a pattern of complaints establishes you as someone who uses the system seriously. Agencies become more responsive to your future requests because they know you will escalate.

### 1.7 The OIA Content Pipeline

Each OIA response flows through a standard process:

1. **Receive and log.** Claude Code updates the tracking system. Flag the response as: full release, partial release, refusal, extension, or "information doesn't exist."

2. **Analyse.** Claude Code reads the documents and extracts: key findings, anything that contradicts government public statements, any data points not previously public, any notable redactions or withholdings.

3. **Draft content.** For each significant finding, Claude Code drafts: a social media post (OIA Reveal format from wave1_content.md), a journalist briefing (one paragraph summary plus the key documents), and a data dashboard update if applicable.

4. **Media distribution.** Send the full documents and summary to the 1-2 most relevant journalists based on the journalist relationship system (section 3 below). Offer exclusivity for 24-48 hours before posting publicly.

5. **Public release.** Post the OIA Reveal content. Include screenshots of the actual documents with key passages highlighted. Always link back to the data dashboard.

6. **Follow-up requests.** Every OIA response suggests new questions. Claude Code drafts follow-up requests based on what was revealed, what was withheld, and what gaps remain.

### 1.8 Filing and Response Timeline

The staggered approach produces a continuous content pipeline:

| Week | Action | Expected Output |
|------|--------|-----------------|
| 1 | File Tranche 1 (requests 1-10) | -- |
| 2 | File Tranche 2 (requests 11-20) | -- |
| 3 | File Tranche 3 (requests 21-30) | -- |
| 4 | Follow up on Tranche 1 acknowledgements | -- |

By week 8, the pipeline should be self-sustaining. Each response generates follow-up requests. Each follow-up yields more specific and more newsworthy material. The first tranche is broad. The second tranche is targeted. The third tranche and beyond are surgical.

## 2. THE PARLIAMENTARY INTELLIGENCE SYSTEM

This system monitors everything that happens in and around Parliament that relates to immigration, population, or the campaign's core issues. The goal is to catch every opportunity to engage with the parliamentary process and to identify every statement by an MP that can be used for accountability.

### 2.1 Hansard Monitoring

Hansard is the official record of parliamentary debates. Everything said in the House is transcribed and published, usually within 24-48 hours.

**What to monitor:** All mentions of: immigration, migration, net migration, visa, population growth, population plan, population strategy, infrastructure capacity, housing demand, migrant workers, AEWV, accredited employer. Also monitor mentions of specific agencies (MBIE immigration, Immigration New Zealand) and specific policy frames (population ponzi, sustainable migration, skills shortage).

**How to monitor:** The parliamentary website (parliament.nz) publishes Hansard transcripts. These are searchable. Claude Code runs a daily check of new Hansard entries against the keyword list. For each match, it extracts: the speaker, the date, the full quote in context, and the subject of debate.

**Processing:** Each flagged statement is logged in the opposition research system (section 5). Claude Code classifies each statement by: party, speaker, topic (immigration levels, migrant workers, infrastructure, housing, etc.), and sentiment (supportive of restriction, supportive of current settings, evasive, critical of another party, etc.). Contradictions with previous statements by the same speaker or party are automatically flagged.

**Practical setup:** The parliament.nz website offers email alerts for sitting day documents, including Order Papers and Hansard. Sign up for all alerts. Supplement with manual checks of the Hansard search function weekly. Claude Code processes any new transcripts in batch.

### 2.2 Select Committee Tracking

Select committees are the primary engagement point for a solo operator in the parliamentary system.

**Committees to monitor:**

- **Education and Workforce Committee:** Handles all immigration legislation. This is the primary committee. Any immigration bill goes here. Also relevant for education infrastructure issues.
- **Finance and Expenditure Committee (FEC):** Examines the Budget. The Budget Policy Statement and the Estimates are opportunities to submit on the fiscal implications of immigration. FEC submissions on the population-infrastructure cost nexus are within scope.
- **Social Services and Community Committee:** Handles housing and social development legislation. Any housing bill with a population growth dimension is an opportunity.
- **Environment Committee:** Handles environmental legislation. Any bill touching on environmental capacity, water infrastructure, or resource management has a population dimension.
- **Transport and Infrastructure Committee:** Handles transport legislation. Infrastructure capacity arguments belong here.

**How to track:** The parliament.nz website lists all current business before each committee, including bills referred and calls for submissions. Subscribe to email updates for each of the five committees above. Claude Code checks the committee pages weekly for new items.

**What to look for:**
- New bills referred to committee (a call for submissions will follow within days)
- Petitions referred to committee
- Estimates hearings (annual budget review, usually May-July)
- Annual reviews of government departments (MBIE, Health NZ, Ministry of Education, etc.)
- Any inquiry initiated by the committee itself

### 2.3 Upcoming Bills and Submission Targets

As of March 2026, these are the types of legislative activity most likely to create submission opportunities in the coming months. Specific bills change, but the categories are predictable.

**Immigration legislation:** Any amendments to the Immigration Act 2009 or changes to immigration instructions. The Education and Workforce Committee handles these. Submit on population planning, the absence of a migration target, and the disconnect between visa approvals and infrastructure investment.

**Budget-related bills:** The Appropriation Bill and any budget legislation go through FEC. Submit on the population dimension of every major spending line: health, education, housing, transport. The frame is always "does this spending account for projected population growth? If not, it's a fiction."

**Housing legislation:** Any bills amending the Residential Tenancies Act, the Urban Development Act, or the Resource Management Act successor legislation. Submit on the demand-side driver of population growth. The frame: "supply-side housing reforms are necessary but insufficient if demand-side population growth is not addressed."

**Infrastructure legislation:** Any bills related to transport investment, water infrastructure (Three Waters reforms or successors), or urban development. Submit on population-driven demand.

**Environmental legislation:** Any bills related to climate targets, emissions budgets, or resource management. Submit on the emissions and environmental impact of population growth.

### 2.4 How to Draft an Effective Select Committee Submission

A select committee submission follows a specific structure. MPs and committee advisors read dozens of submissions per bill. Yours needs to stand out through clarity and data.

**Structure:**

1. **Introduction (1 paragraph).** Who you are. What your submission relates to. Your recommendation in one sentence.

2. **Summary of position (2-3 sentences).** State your core argument. For most submissions, this will be a variation of: "This legislation does not account for the impact of population growth driven by net migration on [the specific topic of the bill]. Without linking [policy area] planning to population projections, the legislation will fail to achieve its stated objectives."

3. **Evidence (the bulk of the submission).** Present your data. OIA findings, Stats NZ data, international comparisons, local impact examples. Every claim must be sourced. Use tables and charts where they add clarity. Keep it under 3,000 words. Committees appreciate concision.

4. **Specific recommendations.** Numbered, actionable. Examples: "That the bill include a requirement for [agency] to publish annual analysis linking [policy area] capacity to population projections." "That a statutory migration advisory body be established to review immigration settings against infrastructure capacity annually."

5. **Request to appear.** Always request to appear in person for oral submission. Oral submissions are more impactful. You are in the room. MPs ask you questions. The exchange goes on the parliamentary record. It can be quoted. And the face-to-face interaction builds relationships with MPs on the committee.

**Practical tips:**
- Submit early in the submission period. Early submissions are more likely to be read carefully.
- Reference specific clauses of the bill, not just the general topic.
- Avoid partisan language. Frame your submission as technical and evidence-based.
- Follow up with the committee clerk to confirm receipt and to confirm your oral submission slot.
- Prepare a 5-minute oral summary for the in-person appearance. Leave time for questions from MPs.

### 2.5 Written Parliamentary Questions

Written parliamentary questions (WPQs) are a tool for getting government responses on the record. You draft them. A sympathetic MP files them.

**How to draft effective questions:**

- Each question must be answerable with a specific fact. "What is the Minister's estimate of..." rather than "Does the Minister agree that..."
- Questions should request data points that reveal the gap between rhetoric and reality.
- Draft them in the exact format Parliament requires: addressed to a specific Minister, numbered, each question self-contained.

**First 10 questions to draft and shop to MPs:**

1. What is the government's target for annual net migration? If no target exists, why not?
2. What is Treasury's estimate of the Crown infrastructure cost per net migrant (including health, education, housing, and transport infrastructure)?
3. How many additional hospital beds has Health NZ planned for as a result of population growth from net migration since 2021?
4. How many additional school places has the Ministry of Education planned for as a result of population growth from net migration since 2021?
5. What net migration assumption underpins the government's current housing supply targets?
6. Has the government received advice recommending the establishment of an independent migration advisory body? If so, when, and what was the government's response?
7. How many Accredited Employer Work Visa holders are currently in New Zealand, by sector and region?
8. What is the median wage paid to AEWV holders, by sector?
9. How many employer accreditations have been revoked for non-compliance since the AEWV scheme's inception?
10. What is the projected net migration figure used in the 2026 Budget forecasts?

**Who to approach:** Start with NZ First MPs (Shane Jones's office is the first call). Frame for NZ First as accountability questions. If NZ First declines, approach ACT MPs with the fiscal framing: "what does this cost the taxpayer?" Labour backbenchers in South Auckland or provincial electorates may file questions framed around housing and worker exploitation.

The operational process: email the MP's electorate office (not Parliament) with the drafted questions in the exact required format, a one-paragraph explanation of why each question matters, and what you expect the answer to reveal. Make it effortless for their staff. A phone call before the email increases the odds. Once filed, questions and answers appear in Hansard within six working days and become permanent public record.

## 3. THE JOURNALIST RELATIONSHIP SYSTEM

This is a lightweight CRM that tracks every interaction with every journalist, what they care about, what you've sent them, what they've used, and how to deepen the relationship from cold outreach to trusted source.

### 3.1 The Tracking System

A single structured file (or a simple spreadsheet) with one row per journalist. Columns:

| Field | Purpose |
|-------|---------|
| Name | Full name |
| Outlet | Current employer |
| Beat | What they cover (politics, housing, economics, etc.) |
| Contact | Email, X handle, other contact details |
| Interest areas | Specific topics they gravitate toward based on published work |
| Preferred angle | How to frame material for them (investigative, policy, political, human interest) |
| Date first contacted | When you first reached out |
| What you sent | Specific material sent, with dates |
| Their response | Did they reply? What did they say? |
| What they used | Did they publish anything based on your material? Link to the article. |
| Last contact date | Most recent interaction |
| Relationship status | Cold / Warm / Active source / Regular contact |
| Notes | Anything relevant: personal interests, preferred communication method, responsiveness |

Claude Code maintains this file and flags when a journalist hasn't been contacted in more than 3 weeks, when a new OIA response matches a journalist's interest areas, and when a journalist publishes something that creates an opportunity for follow-up engagement.

### 3.2 The Relationship Ladder

The progression from nobody to trusted source follows a predictable sequence. Do not skip steps.

**Step 1: Research (before any contact).** Read their last 20-30 articles. Understand what they cover, how they frame stories, what data they use, and what gaps they leave. Claude Code can analyse a journalist's published work and identify their patterns. This research determines whether they are receptive to immigration-infrastructure framing, what angle works for them, and what format of material they prefer.

**Step 2: Passive engagement (weeks 1-2).** Follow them on X. Like and reply to their posts with substantive, data-backed comments. Not flattery. Not "great article." A reply that adds a data point or a perspective they haven't considered. The goal is to become a name they recognise in their notifications as "the person who always has good data."

**Step 3: First direct contact (weeks 2-4).** Send a short, specific email or DM. Not a pitch. An offering. "I've been following your coverage of [topic]. I have some OIA data on [specific thing] that I think adds to the picture. Happy to share if useful. No expectation of coverage." Attach one document or data point. Keep the email under 150 words.

**Step 4: Follow-up with value (ongoing).** Every 2-3 weeks, send something useful. A new OIA finding that matches their beat. A data point that adds context to a story they just published. An international comparison relevant to something they're covering. Never ask for coverage. Always frame as "thought you might find this useful." The relationship compounds: each useful offering increases the likelihood they'll open your next email.

**Step 5: Active source relationship (month 2-3+).** If a journalist uses your material or contacts you for comment, you've crossed the threshold. Now maintain the relationship: respond to their queries quickly, provide context and background (even off the record), and never burn them by giving the same material to a competitor simultaneously without warning.

### 3.4 How to Build from Cold to Trusted Source

The critical rules:

**Never pitch, always provide.** Journalists get dozens of pitches per day. They respond to material they can use. Every interaction should be you giving them something useful with no strings attached.

**Never lie, exaggerate, or spin.** In NZ's small media ecosystem, one dishonest data point will end your credibility permanently. If a number is uncertain, say so. If the data is ambiguous, present both interpretations. Journalists trust sources who are honest about limitations.

**Respond fast.** When a journalist contacts you, reply within hours. Their deadlines are tight. Being responsive is the single easiest way to build a relationship.

**Respect exclusivity.** If you offer a journalist an exclusive, give them 24-48 hours before you share the material elsewhere. Breaking exclusivity will end the relationship.

**Accept that coverage is their decision.** You provide material. They decide whether to use it. Never pressure a journalist to cover something. Never complain about not being credited. The material matters more than the attribution.

**Keep a "what they've used" record.** When a journalist uses your material, log it. Over time, this record tells you which journalists are reliable amplifiers and which are interested but not producing output. Invest more time in the former.

## 4. THE SUPPORTER DATABASE

Even a solo campaign accumulates contacts: people who engage on social media, subscribe to updates, email with questions, share content, or attend events. These contacts become an activation asset when the campaign needs coordinated action (email-your-MP campaigns, select committee submissions, amplifying key content).

### 4.1 What to Capture

For each supporter, track:

| Field | Source |
|-------|--------|

| Email | Signup form |
| Location (region/electorate) | Signup form, self-reported |
| How they found you | Signup form (dropdown: social media, referred by friend, media article, etc.) |
| Engagement level | Automated: subscriber, sharer, commenter, active supporter |
| Skills/interests | Self-reported: "I work in health," "I'm a teacher," "I have data skills" |
| MP/electorate | Derived from location |
| Date joined | Automatic |
| Actions taken | Log of: emails opened, links clicked, submissions filed, MPs emailed |

### 4.2 Free Tools

**Email list: Buttondown (free tier) or Substack.**

Buttondown offers a free tier for up to 100 subscribers, then very low cost. Substack is completely free and has built-in audience-building features (discovery, recommendations). Substack is better for the campaign because it also functions as a publishing platform. The weekly wrap (Pillar 7 from wave1_content.md) can be published on Substack, which builds the email list automatically.

Recommendation: Use Substack as the primary email and publishing platform. Every piece of long-form content (weekly wrap, scorecard updates, OIA analysis) lives on Substack. Social media posts drive traffic to Substack, which captures email subscribers.

**Contact management: Airtable (free tier) or a structured Google Sheet.**

Airtable's free tier supports up to 1,000 records with multiple views, filters, and linked records. This is sufficient for the first several months. Use it to track the full supporter database with the fields above.

If Airtable feels like overkill, a Google Sheet with the same fields works. Claude Code can process and update it.

**Social media analytics: Native platform analytics (free).**

X Analytics, TikTok Analytics, Instagram Insights, and Facebook Page Insights are all free with a business or creator account. These show you who your audience is, what content performs, and when people are online.

**Action tools: Google Forms (free) for surveys and submissions; "Email your MP" functionality via a simple web page on GitHub Pages.**

Build a minimal page that: lets users enter their electorate or postcode, returns their MP's name and email address (using publicly available data from parliament.nz), and provides a pre-drafted email template they can copy-paste or send directly. Claude Code can build this page in an afternoon. It does not need to be fancy. It needs to work.

### 4.3 Capturing Contacts

The sign-up funnel runs through two primary channels:

**From social media:** Every post includes a link to Substack in the bio or profile. The weekly wrap is the primary conversion mechanism: "Want this in your inbox every Friday? Subscribe at [Substack link]." Do not push the subscribe CTA on every post. Save it for the weekly wrap and for breakout moments when engagement is high.

**From OIA Reveal and data content:** When a post gets strong engagement, follow up with: "If you want to stay across this, I do a weekly wrap covering all the OIA findings and data updates. Link in bio." The implicit offer is "I do the work, you get the summary." This converts the casually interested into subscribers.

**From events and in-person contacts:** If you attend select committee hearings, candidate meetings, or public events, have a simple card or QR code that links to the Substack. "Want to follow the campaign? Scan this." Low friction.

### 4.4 Activating Supporters

The supporter list becomes powerful at specific moments. These are the activation triggers:

**Select committee submissions.** When a relevant bill is before a select committee, email the list: "The [bill name] is before the [committee]. Here's why it matters for population planning. Here's a template submission you can adapt and file. The deadline is [date]." Include a pre-drafted submission template and clear instructions. Even 20-30 submissions from individuals across the country make a committee sit up. The volume matters.

**Email your MP campaigns.** When a specific parliamentary question or issue is live, email the list: "Ask your MP this one question: [question]. Here's how to find your MP's email. Here's a template you can use." The "email your MP" page on the website makes this one-click. Target these campaigns to specific moments when the pressure matters: before a vote, during a policy announcement, or in response to a refusal to answer.

**Content amplification.** When a critical piece of content (a scorecard, a major OIA reveal, an accountability question) is published, email the list: "This just went up. If you think it matters, share it." The coordinated sharing pushes content through the algorithm's engagement thresholds, making it visible to a much larger audience.

**Petition signatures.** If a petition to Parliament is drafted (as suggested in wave1_nz_ops.md), the email list is the first source of signatures.

### 4.5 Segmentation

Even a simple list benefits from segmentation:

**By electorate.** When running an "email your MP" campaign, you only need to activate supporters in the relevant electorates. Sending a generic blast is less effective than "Your MP is [Name]. They haven't answered the question. Here's how to ask them directly."

**By engagement level.** Some subscribers open every email. Some haven't opened one in months. Focus activation asks on the engaged segment. The disengaged segment still receives the weekly wrap (which may re-engage them) but is not asked to take action.

**By skills.** If someone has self-identified as a health worker, teacher, or data analyst, they can be activated for specific tasks: a health worker's select committee submission on hospital capacity carries more weight than a generic submission.

## 5. THE OPPOSITION RESEARCH SYSTEM

This system tracks what every party and every relevant MP says about immigration, population, and related topics. It identifies contradictions, tracks shifting positions, and provides the raw material for accountability content.

### 5.1 What to Track

For every statement captured, log:

| Field | Detail |
|-------|--------|
| Date | When the statement was made |
| Speaker | Full name and party |
| Role | Minister of X, spokesperson for Y, backbencher, party leader |
| Source | Hansard, media article, social media post, press release, speech |
| Source URL | Link to the original |
| Full quote | The exact words, in context |
| Topic | Immigration levels / migrant workers / housing-population / infrastructure / fiscal cost / environmental / Treaty |
| Stance | Supports restriction / supports current settings / evasive / silent / contradicts previous position |
| Contradicts | Link to any previous statement by the same speaker or party that is inconsistent |
| Campaign use | How this statement can be used: accountability content, scorecard update, journalist briefing, WPQ draft |

### 5.2 Sources to Monitor

**Hansard (daily when Parliament sits).** Every oral question, every debate speech, every committee report. Claude Code processes new Hansard entries daily against keyword lists.

**Media articles (daily).** Monitor NZ Herald, Stuff, RNZ, Newsroom, Interest.co.nz, and The Kaka for any article quoting a politician on immigration or related topics. Claude Code can check RSS feeds or website sections for new articles.

**Social media (daily).** Monitor the X accounts of all party leaders and immigration/housing spokespeople. Also monitor relevant hashtags and keyword searches. Key accounts to follow: all MPs from the journalist map's "sympathetic MPs" lists, plus all party leaders and relevant spokespeople.

**Party websites and press releases (weekly).** Check each party's website for policy announcements, press releases, and speeches. These often contain statements that aren't picked up by media.

**Select committee transcripts (as published).** When a select committee hears oral submissions or questions officials, the transcript is published. These often contain revealing statements by MPs.

### 5.3 How Claude Code Processes and Classifies Statements

The workflow runs in batch, once daily during sitting weeks and twice weekly during recess:

1. **Collect.** Claude Code checks all sources for new material matching the keyword list: immigration, migration, net migration, visa, population growth, population plan, housing demand, infrastructure capacity, migrant workers, AEWV, plus all party-specific terms.

2. **Extract.** For each match, extract the structured data described in section 5.1.

3. **Classify.** Assign topic and stance codes. The stance classification uses a simple taxonomy:
   - **RESTRICT:** Speaker advocates lower migration, tighter controls, or a numerical cap
   - **MAINTAIN:** Speaker defends current settings or opposes restriction
   - **EVADE:** Speaker avoids stating a position despite being asked
   - **SILENT:** Speaker/party has not addressed the topic (tracked by absence)
   - **CONTRADICT:** Speaker's statement is inconsistent with a previous logged statement

4. **Flag contradictions.** For each new statement, Claude Code compares it against all previous statements by the same speaker and the same party. If the stance has changed, the topic framing has shifted, or specific claims are inconsistent, it flags the pair for review.

5. **Generate output.** Flagged contradictions generate draft accountability content: "In [month], [MP] said [quote 1]. This week, [MP] said [quote 2]. Which is it?" These drafts go into the content pipeline for review before publishing.

### 5.4 Contradiction Detection

Contradictions are the highest-value output of the opposition research system. There are several types:

**Numerical contradictions.** MP claims a number in one context, a different number in another. "Minister said visa numbers were 'stable' in March, but MBIE data released in April showed a 30% increase."

**Positional contradictions.** MP advocates one position to one audience and a different position to another. "Party X told the business community they support high migration for skills. Party X told suburban voters they'd 'reduce the pressure.' Both claims are on the record."

**Promise-vs-delivery contradictions.** Party promised something and hasn't delivered. "NZ First promised to cut net migration to 10,000-15,000. They've been in government for [X] months. Net migration is [Y]."

**Internal party contradictions.** Different MPs from the same party take different positions. This is particularly useful for Labour (union wing vs progressive wing) and National (business wing vs suburban voter wing).

**Cross-temporal contradictions.** Party's current position contradicts their position in opposition, or vice versa. "National criticised Labour's migration numbers in 2023. National's own numbers are now higher."

### 5.5 The Party Position Database

In addition to individual statements, maintain a summary of each party's current position:

| Party | Stated target | Housing link | Infrastructure link | Advisory body | Key spokesperson | Last updated |
|-------|---------------|-------------|---------------------|--------------|-------------------|-------------|
| National | None stated | None | None | None | [Immigration spokesperson] | [date] |
| Labour | None stated | None | None | None | [Immigration spokesperson] | [date] |
| NZ First | "Significantly reduced" (no number) | Partial | None | Proposed (no detail) | Shane Jones | [date] |
| ACT | None stated | None | None | None | [spokesperson] | [date] |
| Greens | None stated | None | None | None | [spokesperson] | [date] |
| TPM | None stated | None | None | None | [spokesperson] | [date] |

This table feeds directly into the scorecard content (wave1_content.md). It is updated every time a party makes a relevant statement. The "None stated" entries are the campaign's most powerful data points, because they demonstrate the planning vacuum.

### 5.6 Turning Opposition Research into Content

The system generates four types of content:

**Contradiction compilations.** Side-by-side quotes showing inconsistency. Published as image cards (screenshot of quote 1 next to quote 2 with dates) and short video (reading both quotes aloud). These are among the most shareable content formats in political social media.

**Scorecard updates.** When a party changes position or makes a new commitment, update the scorecard. The update itself is content: "We've updated the Immigration Scorecard. [Party] now gets a [grade] because [reason]."

**Journalist briefings.** When a contradiction is identified, package it and send to relevant journalists: "You may want to ask [Minister] about this. In March they said X. Their own department's data shows Y."

**Parliamentary question drafts.** Contradictions between ministerial statements and departmental data are perfect for written parliamentary questions. Draft the question and shop it to a sympathetic MP.

## 6. SYSTEM INTEGRATION

These five systems are not independent. They feed into each other continuously.

**OIA pipeline feeds opposition research.** When an OIA response reveals data that contradicts a ministerial statement, the opposition research system flags it and generates content.

**Opposition research feeds the journalist system.** When a contradiction is identified, the journalist CRM identifies which journalist would be most interested (based on beat and interest areas) and the material is sent to them.

**Parliamentary intelligence feeds the OIA pipeline.** When an MP makes a claim in Hansard about immigration numbers or policy, the OIA system generates a targeted request to verify or challenge the claim.

**Supporter database activates on parliamentary triggers.** When a bill is referred to a select committee, the supporter list is activated for submissions. When a parliamentary question reveals an evasive answer, the supporter list is activated for "email your MP" pressure.

**Journalist relationships amplify everything.** Every system produces material that can be shared with journalists. The journalist CRM ensures the right material goes to the right journalist at the right time.

### Daily Workflow

The daily operational rhythm, managed by Claude Code with human review:

**Morning (30 minutes):**
- Check Hansard for new entries (sitting days). Process and classify.
- Check media for new articles mentioning immigration/population. Process and log.
- Check OIA tracking for approaching deadlines, overdue responses, or responses received.
- Review any flagged contradictions from overnight processing.

**Midday (30 minutes):**
- Draft and schedule the day's content based on what the morning review revealed.
- Send any journalist briefings triggered by new findings.
- File any follow-up OIA requests triggered by new information.

**Afternoon (30 minutes):**
- Engage on social media: respond to replies, engage with journalists' posts, monitor what's trending in NZ politics.
- Update the supporter database with any new signups.
- Update the party position database if any party has made a relevant statement.

**Friday (additional 1 hour):**
- Compile the weekly wrap from the week's findings.
- Update the scorecard if warranted.
- Review the journalist CRM and plan next week's outreach.
- File new OIA requests based on the week's findings.

Total daily time commitment: approximately 90 minutes of human time, plus Claude Code processing in the background. This is sustainable for a solo operator and produces output equivalent to a small team.

## 7. TECHNICAL IMPLEMENTATION

All systems are built with free tools and Claude Code. No paid software is required.

**OIA tracking:** A structured JSON file or SQLite database managed by Claude Code. Fields: request ID, agency, subject, date filed, deadline, status, response summary, follow-up actions, content generated. Claude Code reads and writes this file, generates reminders, and drafts follow-up emails.

**Opposition research database:** Same approach. A structured file with one entry per statement. Claude Code processes new entries, classifies them, and runs contradiction checks against the full database.

**Journalist CRM:** A structured file or spreadsheet (Google Sheets for accessibility, or a local file). Claude Code updates it after each interaction and flags opportunities.

**Supporter database:** Substack handles email subscribers natively. Airtable (free tier) or Google Sheets handles the extended contact data. The "email your MP" tool lives on GitHub Pages as a simple HTML/JavaScript page.

**Data dashboard:** Hosted on GitHub Pages (free). Built with HTML, JavaScript, and a charting library (Chart.js or similar). Claude Code builds and updates it. Data sourced from Stats NZ, MBIE, and OIA responses.

**Monitoring scripts:** Claude Code can process RSS feeds, check websites, and parse Hansard transcripts on demand. Where automated daily checks are needed, simple cron jobs or scheduled scripts can be set up on any always-on machine (even a Raspberry Pi or a free-tier cloud instance).

The entire technical stack costs nothing except the domain name for the dashboard (optional; a github.io subdomain works fine) and the human time to operate it.
